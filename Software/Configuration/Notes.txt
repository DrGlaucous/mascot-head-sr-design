**If OS gets corrupted again**

must copy cmdline.txt and config.txt onto the SDcard (with those names exactly). 
The good versions are in this folder, just remove the "CM4_" prefix when you copy to SDcard.

after flashing a new image onto the pi and copying the config files into the SDcard, run the pi and 
talk to it using HDMI screen and a USB keyboard. 

run this command on the Pi: sudo systemctl enable getty@ttyGS0.service
Check that it worked with: systemctl is-enabled getty@ttyGS0.service (should return "enabled")

after that, you can talk to it through serial like usual.
